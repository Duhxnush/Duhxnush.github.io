Overview

What This Guide Covers

This guide explains the full CIB Bulk Upload automation workflow — all steps from login and file preparation through to multi-user authorization, details validation, post-processing, and secure logout.

Design Principles

Core Engineering Principles

⚙️
Modularity & Reusability
Functions are discrete and reusable across payment types.
📂
Dynamic File Handling
Files are renamed and updated with unique payment data to ensure validation and avoid duplicates.
✔️
Validation-Oriented
Verifications at key workflow points ensure correctness at every stage.
🔍
Traceability
Runtime variables, file paths, and identifiers are stored for logging and debugging.
Workflow

Detailed Phases

01
Login — Initiator
Auth ▶
Function: CIB_Login (v1.0)
Authenticates the initiating user to provide secure access for bulk upload operations.
User: Initiator v1.0
02
Navigation
Setup ▶
Function: CIB_Bulk Uploads_Navigation (v1.0)
Navigates to the bulk upload test case screen within the application.
User: Initiator v1.0
03
File Preparation
Data ▶
Function: CIB_Bulk Uploads_Download and Update SDMC File (v1.0)
Downloads the template, renames the file, and updates XLSX / CSV / TXT with unique payment data.
Formats: XLSX · CSV · TXT User: Initiator
04
Bulk Upload Process
Core ▶
Functions: Domestic/Internal Uploads, Payment Review, Receipt Generation, Bulk Summary
Fills payment details, validates the review screen, generates receipts, and checks the bulk summary.
User: Initiator Receipt Generated
05
Authorization Process
Multi-User ▶
  • Submit for Authorization — Self Auth (Initiator)
  • Verifier login → Pending Tasks Approval/Rejection → Bulk Summary → Logout
  • Releaser login → Pending Tasks Approval/Rejection → Bulk Summary → Details Validation → Logout
Users: Initiator · Verifier · Releaser
06
Post-Processing
Validation ▶
Assigns the correct debit type and performs country-specific validation after final approval.
Country-Specific Logic Debit Type Assignment
07
Logout
Cleanup ▶
Logs out all users to ensure session security and environment cleanup.
All Users Session Terminated

Best Practices

Recommendations

👤
Map each login session to the correct workflow user role.
✅
Run Details Validation only after final Releaser approval.
🔐
Ensure pending task approvals/rejections and logouts are completed per user session.
🗂️
Use unique data in uploaded files to support accurate validation.
🧩
Keep functions modular and reusable across payment types.
🔎
Trace file paths and runtime variables to simplify debugging.
