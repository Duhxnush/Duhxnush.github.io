Category 01
Scripting Issues
Mapping Error
Wrong Status Mapping in Bulk Summary
The script maps an incorrect status value in the bulk summary step, causing validation mismatches and test failures.
Validation Error
Wrong Mapping or Text Validations in Receipt / Submit for Authorization
Incorrect field mappings or mismatched text assertions in the receipt or authorization submission step break downstream checks.
Test Data Error
Wrong Test Data Passed to Bulk Summary
Incorrect or stale test data is supplied to the bulk summary function, leading to assertion failures and unpredictable results.
Encoding Error
Conversion Failed — Special Characters in Random String
Randomly generated strings containing special characters break the file conversion step, causing runtime exceptions.
⚠️
Category 02
Application Issues
Rejection
Payments Being Rejected in Summary
Payments that should pass are rejected at the summary stage due to application-side validation bugs or configuration issues.
Conversion
Conversion Failure — Application-Side
The application fails to convert the uploaded file due to internal processing errors unrelated to the file content itself.
Performance
Excessive Loading
Pages or processing steps take an abnormally long time to respond, causing timeouts and cascading test failures.
Stuck State
Payments Stuck in "Success" or "In Process"
Payments remain indefinitely in an intermediate state, preventing completion of the workflow and blocking downstream steps.
Misc Errors
Other Errors & Invalid Rejections
Any other unexpected application errors, or payments being rejected when they should not be — indicative of defects requiring investigation.
🔧
Resolution Guide
Quick Fixes
01
Replicate via the Same Flow in the Report
Use the same execution flow documented in the report and attempt to reproduce the failure manually. This confirms whether the issue is a genuine defect or a transient/environment-specific problem before escalating.
Defect Confirmation
02
Fix Test Data Mapping
For test data mapping failures, verify the correct user is mapped in the login step, that the correct status is used for bulk summary assertions, and that all field values in the Details Validation function match the expected data precisely.
Data Mapping
03
Compare Screenshots Against Expected Baseline
Refer to the captured screenshot from the failed run and compare it side-by-side with the expected screenshot. Visual differences pinpoint where the UI diverges from the expected state, narrowing down the root cause quickly.
Visual Validation
04
Inspect the Logs for Detailed Failure Reason
Refer to the execution logs to find the exact error message, stack trace, or assertion output. Logs provide the ground truth for what the system encountered, especially for application-side issues that are not visible in screenshots.
Log Analysis
💡

Tip: Always categorise a failure as either a scripting issue or an application issue before applying a fix. Scripting issues can be resolved in the automation codebase; application issues must be raised as defects with the development team and tracked through the standard defect lifecycle.
